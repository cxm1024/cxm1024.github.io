ABC176F Brave CHAIN题解

发表于 2023-01-19 分类于 题解

题目传送门

题意：有 3n3n3n 张卡片，每张有一个 1∼n1\sim n1∼n 的数字。每次可以将最左边的 555 张卡片任意排列，删掉前 333 张，如果这三张数字相等则得一分；最后剩下的三张如果相等也的一分。求最大总得分。

模拟一下这个过程可以发现，相当于你有两张“手牌”，每次新加入三张，你从五张中扔掉三张，还是剩下两张。于是可以考虑一个 DP 状态：f[i][x][y]f[i][x][y]f[i][x][y] 表示进行 iii 次操作，操作后手上剩下的牌为 x,yx,yx,y 的最大得分。转移也不难，枚举下一次新加进来三张后如何选即可。时空复杂度 O(n3)O(n^3)O(n3)。

仔细想一下，会发现对于相同的 x,yx,yx,y，f[i][x][y]f[i][x][y]f[i][x][y] 随着 iii 的增加永远不会减少，因为最差的情况无非是：每新加三张就把那三张扔掉，永远让 x,yx,yx,y 为手牌，得分永远不会减少。这启发我们考虑 f[i]→f[i+1]f[i]\to f[i+1]f[i]→f[i+1] 的变化。

首先，新加入的三张牌 a,b,ca,b,ca,b,c 与 x,yx,yx,y 是多少无关，只与当前考虑到哪张牌有关。如果 a=b=ca=b=ca=b=c，不管 x,yx,yx,y 是多少，一定直接把它扔掉，贪心地立刻得分。正确性容易证明，这里不赘述。接下来考虑不全相等的情况。

如果用上面说的那样“消极”做法，新加入三张牌就扔掉，f[i+1][x][y]=f[i][x][y]f[i+1][x][y]=f[i][x][y]f[i+1][x][y]=f[i][x][y]，这是下限，可以直接从上一维过来，所以只需要考虑哪些 x,yx,yx,y 会发生改变即可。与消极做法相反，要想改变一定要让操作后手牌中有新牌。我们先考虑如何扔牌能造成贡献，再考虑不能造成贡献的情况。

假如三张牌中有两张相等（这里假设是 a=ba=ba=b），那么只要 x,yx,yx,y 中有一个与 a,ba,ba,b 相等的牌即可凑起来扔掉，而手牌中的另一张牌与 ccc 组成新的手牌。此时，我们可以枚举那个“另一张牌”，假设为 jjj，那么有 f[i][a][j]+1→f[i+1][j][c]f[i][a][j]+1\to f[i+1][j][c]f[i][a][j]+1→f[i+1][j][c]（这里暂时不考虑两维的手牌的顺序，实现的时候将 j,cj,cj,c 和 c,jc,jc,j 都更改即可）。接下来，考虑两张手牌相等，与另一张相等的新牌配对，可以直接枚举与哪张新牌相等，假设为 aaa，则有 f[i][a][a]+1→f[i+1][b][c]f[i][a][a]+1\to f[i+1][b][c]f[i][a][a]+1→f[i+1][b][c]。于是，造成新贡献的考虑完成。

接着考虑不造成新贡献的转移：由于前面说过，新的手牌中必须留新牌，我们可以枚举留哪张新牌，再枚举另一张牌是什么。假设留的新牌是 aaa，另一张牌是 jjj，那么如果 j=bj=bj=b 或 j=cj=cj=c，即新的手牌纯用新牌组成，此时旧手牌 x,yx,yx,y 无论是什么都可行。可以维护所有 DP 值的 max⁡\maxmax。如果 jjj 不和 b,cb,cb,c 相等，则旧手牌必须有 jjj，我们维护一个数组 g[j]g[j]g[j] 表示手牌包含 jjj 的所有 DP 值的 max⁡\maxmax 即可。

以上两种转移的次数都是 O(n)O(n)O(n) 的，接下来说回“继承”：加入三张牌直接扔掉，手牌不变。如果直接枚举 x,yx,yx,y，更新 f[i+1][x][y]=f[i][x][y]f[i+1][x][y]=f[i][x][y]f[i+1][x][y]=f[i][x][y] 的话，复杂度为 n2n^2n2，所以这里我们偷个懒，省掉第一维 iii，直接在旧的 fff 数组上做修改即可。注意修改必须考虑完所有转移后整体修改，因为原来的转移是 f[i][x][y]→f[i+1][x′][y′]f[i][x][y]\to f[i+1][x'][y']f[i][x][y]→f[i+1][x′][y′]，而不能 f[i+1][x][y]→f[i+1][x′][y′]f[i+1][x][y]\to f[i+1][x'][y']f[i+1][x][y]→f[i+1][x′][y′]。这可以通过 vector 存储 DP 转移实现。具体可以看代码。

SAM 是干什么的

SAM，后缀自动机，顾名思义，是后缀的自动机（划掉）。可以理解为一个升级版的 trie 树，其中 trie 树内保存某字符串的所有后缀。例如，对于字符串 s=aababcs=\texttt{aababc}s=aababc，其 trie 树如下：

用 trie 树存字符串的每个后缀的其中一个用处为，可以快速地找到一个字符串 ttt 是否在 sss 中出现（当然不止这一个用途，最后会讲应用）。

容易发现，这种 trie 树包含了很多浪费，如下图：

紫色的部分和蓝色的部分结构完全一样，所以可以从根节点直接连一个 b\texttt{b}b 的边，指向 ab\texttt{ab}ab 的节点。SAM 就可以理解成没有任何浪费的 trie 树。想法很自然，但如何形式化的完善这一想法呢？

endpos 集合与 parent tree

定义一个函数 endpos(t)\mathrm{endpos}(t)endpos(t)，表示一个字符串 ttt 在 sss 中所有出现的结束位置集合。如，对于上面的例子 s=aababcs=\texttt{aababc}s=aababc，endpos(ab)={3,5}\mathrm{endpos}(\texttt{ab})=\{3,5\}endpos(ab)={3,5}。

另一种直观的理解方式为，我当前从字符串 sss 上某个位置开始走，知道我依次经过了 ttt 的字符串，那么 endpos(t)\mathrm{endpos}(t)endpos(t) 会告诉我，当前我可能处在哪些位置。

我们发现，对于上面例子 s=aababcs=\texttt{aababc}s=aababc 来说，endpos(b)=endpos(ab)={3,5}\mathrm{endpos}(\texttt{b})=\mathrm{endpos}(\texttt{ab})=\{3,5\}endpos(b)=endpos(ab)={3,5}，那么在上面的 trie 树上，b\texttt{b}b 和 ab\texttt{ab}ab 的节点子树完全相同。感性理解，我走了 b\texttt{b}b 和 ab\texttt{ab}ab 后可能所处的位置完全相同，那么后面能怎么走自然也完全相同。

于是我们可以得出一个结论：在 SAM 中，endpos\mathrm{endpos}endpos 相同的字符串可以共用一个节点。形式化地，将 sss 的所有子串 ttt 划分成若干个“endpos\mathrm{endpos}endpos 等价类”，则每个等价类与 SAM 上的节点一一对应。

我们显然不能暴力地去找每一个等价类，这就引出了一个新概念：parent tree。它建立了所有 endpos\mathrm{endpos}endpos 等价类之间的关系。

还是以 s=aababcs=\texttt{aababc}s=aababc 来说。如果我当前走了 a\texttt{a}a，那么我当前可能在 {1,2,4}\{1,2,4\}{1,2,4}。但如果我当前走了 ba\texttt{ba}ba，我当前只可能在 {4}\{4\}{4}。此时，在一个字符串前加上一个新的字符，可能会产生新的限制，从而将集合分裂出去。parent tree 上的边表示的就是这种分裂。

对于 s=aababcs=\texttt{aababc}s=aababc，parent tree 的形态如下：

这里说一下对于这张图的感性理解：假如我走了一步，经过 b\texttt{b}b，我当前可以在 {3,5}\{3,5\}{3,5}。如果我加强了限制，走了两步，为 ab\texttt{ab}ab，那么我当前仍可能在 {3,5}\{3,5\}{3,5}。如果我走了三步，为 bab\texttt{bab}bab，那么我只能在 {5}\{5\}{5}。同理，如果走了四步，为 abab\texttt{abab}abab，同样只能在 {5}\{5\}{5}。

通过感性理解可以发现，parent tree 有几个非常重要的性质：

  1. 一个点代表的若干个字符串为最长的字符串的后缀，且长度连续（不可能出现代表 d,bcd,abcd\texttt{d,bcd,abcd}d,bcd,abcd 而不代表 cd\texttt{cd}cd。
  2. 一个节点的祖先为该节点字符串的后缀，且依次往上遍历可以遍历到所有后缀。
  3. 由性质 2 推出，一个节点的最短字符串长度为，父亲节点的最长长度+1+1+1。

接下来考虑 parent tree 的节点数（即不同的等价类个数，也就是 SAM 的节点数）。由于叶子节点的集合元素个数为 111，每个节点的元素个数都至少为儿子之和，最终根节点元素个数为 nnn，所以总节点数不会超过 2n−12n-12n−1（易证，这里不赘述）。

构建 SAM

来到重头戏了！在这里，我们采取动态构造的方法，依次加入每个字符，并同时维护 parent tree 和 SAM。因为两者的节点一一对应，所以我们在同一个结构体里维护：

其中 sam[u].fa 表示 uuu 在 parent tree 上的父亲，sam[u].nxt[ch] 表示 uuu 在 SAM 上经过 chchch 到达的节点，sam[u].len 表示 uuu 所表示的最长字符串的长度。最短长度即为 sam[sam[u].fa].len+1。

这里先把算法流程过一遍，有一个框架，后面再详细说明。

首先有根节点，这里设为 111 号节点。

加入字符的过程中，维护当前整个串对应的节点 lstlstlst。

加入一个字符 chchch 时，新建一个节点 curcurcur 表示整个串（也可能表示了其他串，但整串是最长的）。

从 lstlstlst 开始在 parent tree 上往上爬，直到爬到一个节点 ppp 已经存在 chchch 的出边。在这之前，每个节点都没有 chchch 的出边，所以新建一个 chchch 的出边指向 curcurcur。

如果爬完了 parent tree，每个节点都没有 chchch 的出边，则将 sam[cur].fa 设为根。否则，设碰到的节点为 p→chqp\xrightarrow{ch}qpch​q，作如下判断：

  1. 如果 sam[p].len+1==sam[q].len，那么将 sam[cur].fa 设为 qqq，结束。
  2. 否则，新建一个节点 rrr，信息与 qqq 完全相同，但 sam[r].len=sam[sam[q].fa].len+1，并将 sam[q].fa 和 sam[cur].fa 改为 rrr。接下来，让 ppp 继续往上爬，对于每个 p→chqp\xrightarrow{ch}qpch​q 改为 p→chrp\xrightarrow{ch}rpch​r。

最后将当前整个串的节点 lst 改为 cur。

看完肯定非常晕，但没关系，我们接下来通过一个例子，将每一步的原理解释清楚。

对于 s=aababbs=\texttt{aababb}s=aababb，我们模拟一遍上述过程。前面的两个字符不具有代表性，我们从第三个开始。

左图为加入前两个字符 aa\texttt{aa}aa 后的形态，右图新加入了字符 b\texttt{b}b。观察左图，根据前面说的过程，找到原先整个串对应的节点 222，顺着黑色的 tree 边往上爬。容易发现，一直到根节点，每个节点都不存在 b\texttt{b}b 的出边。于是将每个点走 bbb 的出边连向新点 333。

我们当然不只是模拟过程，接下来进行解释：

根据 tree 的性质，在原图上往上爬一定能遍历原串所有的后缀，且是从长往短遍历。我们希望让新图也能存下新串的所有后缀，而新串的后缀等于原串后缀+字符 b\texttt{b}b。所以，如果一个原串后缀没有字符 b\texttt{b}b 的出边，就说明对应的这个新串后缀没有被保存过，必须向新点连 b\texttt{b}b 的边。连边的意思是，让原串后缀点+字符 b\texttt{b}b 得到的新串后缀，归属这个新点。

这里一直到根节点都不存在 b\texttt{b}b 的出边，也就说明原串中不存在任何一个新串的后缀。然而，新节点的 fa 必须为新串的后缀，所以只能连表示空串的根节点了。

左图为加入 aab\texttt{aab}aab 后的形态，右图为新加入字符 a\texttt{a}a 后的形态。观察左图，同样模拟一开始说的过程：找到整个串对应的节点 333，没有 a\texttt{a}a 的出边，所以连到新节点 444；往上爬到 000，发现 000 已经有了一条 aaa 的出边指向 111，且 111 的 lenlenlen 等于 000 的 len+1len+1len+1，所以将新点 444 的 fa 设为 111。

事情就是从这里开始奇怪的。这里的过程明显比上面难懂，解释如下：

考虑节点 333 表示的字符串有哪些。容易发现，在原串的所有后缀中，aab,ab,b\texttt{aab,ab,b}aab,ab,b 归到节点 333。由于能归到同一个节点，它们同属一个等价类，也就是对后面的影响可以看作等价的。这个节点不存在 a\texttt{a}a 的出边，意味着原串不存在 aaba,aba,ba\texttt{aaba,aba,ba}aaba,aba,ba，但这些都是新串的后缀。那么我们将这个节点连到 444，就意味着让 aaba,aba,ba\texttt{aaba,aba,ba}aaba,aba,ba 都归属到节点 444。但是新串的后缀不止有这三个，还有更短的 a\texttt{a}a 等，于是继续往上跳。

跳到了节点 000，它表示的字符串为空串，通过字符 a\texttt{a}a 的出边自然就是 a\texttt{a}a。这里已经有节点 111 表示 a\texttt{a}a 了，说明新串的后缀 a\texttt{a}a 已经在原串中出现过，那么就不需要让 a\texttt{a}a 来归属 444 了，只需归属 111 即可。为了让 444 往上跳能跳到所有后缀，我们让它的 fa 设为 111。如果有更短的后缀，自然也在原串中出现过，从 111 往上跳也能遍历到，所以不需要再添加新边了。

读到这里，可能还有一个疑问，为什么要求 111 的 lenlenlen 等于 000 的 len+1len+1len+1 呢？这个问题我们将在下面对比解释。

左图为添加 aaba\texttt{aaba}aaba 后的形态，右图新添加了字符 b\texttt{b}b。这次我们不模拟了，直接开始讲解（因为太麻烦了）。

观察左图，找到原串的节点 444，它代表的字符串有 aaba,aba,ba\texttt{aaba,aba,ba}aaba,aba,ba，这些都是原串的后缀，而它们不存在 b\texttt{b}b 的出边，即原串中不存在 aabab,abab,bab\texttt{aabab,abab,bab}aabab,abab,bab，所以要连到新点 555 上。此时新点 555 表示的字符串有 aabab,abab,bab\texttt{aabab,abab,bab}aabab,abab,bab。

接着沿着 tree 往上爬，找到节点 111，它代表的字符串为 a\texttt{a}a。这个节点存在 b\texttt{b}b 的出边，指向 333，这意味着原串中已经存在新串的后缀 ab\texttt{ab}ab，且归属节点 333。

如果我们像上面一样，直接把新点 555 的 fa 设为 333，即节点 333 为新串的后缀，就会产生问题： 333 所代表的另一个字符串 aab\texttt{aab}aab 并非新串后缀。

因为，333 的 lenlenlen 并不是 111 的 len+1len+1len+1，这意味着 333 代表的最长串并非由 111 转移而来。如果 333 表示的某个串比 ab\texttt{ab}ab 更长，则其一定不是新串后缀。这是因为，比 ab\texttt{ab}ab 更长的新串后缀如 bab\texttt{bab}bab 等，已经确定在原串中没有出现，所以才连到了新点 555。

原来，在原串中 ab\texttt{ab}ab 和 aab\texttt{aab}aab 之所以能共用一个节点 333，是因为历史的局限性，我们以为这两个串是等价的。但加上新字符后，我们发现，ab\texttt{ab}ab 是新串后缀而 aab\texttt{aab}aab 不是，所以这两个必须区分开。为此，我们专门开一个新节点 rrr，只存这个 ab\texttt{ab}ab，而剥夺节点 333 对 ab\texttt{ab}ab 的占有权。

由于以前这两个等价，所以 rrr 应该与节点 333 有相同的出边，但对于所有原串的更短的后缀 xxx，如果 x→b3x\xrightarrow{\texttt{b}}3xb​3，则更改为 x→brx\xrightarrow{\texttt{b}}rxb​r。这是因为，xxx 存的一定为 a\texttt{a}a 的后缀，添加一个字符 b\texttt{b}b 才到 ab\texttt{ab}ab，而现在 rrr 才是真的 ab\texttt{ab}ab，所以要这样连。对于那些连到 3:aab3:\texttt{aab}3:aab 的点，如 2:aa2:\texttt{aa}2:aa，一定不是原串的后缀，不会被更改。

此时，还要将 333 的 fa 设为 rrr（因为 rrr 显然是 333 的后缀），且 rrr 的 fa 设为 333 的原 fa。这样，我们才能在 tree 上正确的遍历这些串的后缀。最后的最后，别忘了将新点的 fa 设为 rrr。

现在可以解释第二节最后的问题了：为什么 000 的 len+1len+1len+1 等于 111 的 lenlenlen，就不需要这么麻烦？因为此时 111 代表的字符串总共就只有一个，不会出现因历史局限性而混淆两个串的情况，是合法的新串后缀。

完结撒花！ 如果有哪些部分没看懂或不清楚，强烈建议先对照图和算法流程模拟一遍，再看我的解析。个人认为绝对能理解（蜜汁自信）

应用

基础应用可以直接看 pecco 大佬的原文，讲得比较明白。这里着重说一下较难理解的最长公共子串。

要求 sss 和 ttt 的最长公共子串，可以对 sss 建 SAM，从左到右扫 ttt 的每一位，算以这一位结尾的最长公共子串所在的节点。每次新加入一位时，如果可以直接转移则转，如果没有这条转移边，就退而求其次，跳到 fa 节点试图转移（因为 fa 是本质不同的最长后缀），直到有转移边为止。

如果要求 s1,s2,⋯ ,sns_1,s_2,\cdots,s_ns1​,s2​,⋯,sn​ 的最长公共子串，可以对 s1s_1s1​ 建 SAM，把 s2∼sns_2\sim s_ns2​∼sn​ 都跑一遍，对每个节点算出跑到这个节点的最短长度（因为要公共子串），最后取 max⁡\maxmax。注意，跑到一个节点相当于也跑到了它的 tree 上的祖先（因为祖先是后缀），所以要树形 DP 更新一遍再统计答案。

更多神仙应用可参见 OI-wiki。
